# Workflow: Product root

Procedure for creating or revising **`docs/product/README.md`**. Paths: [`../assets/CONVENTIONS.md`](../assets/CONVENTIONS.md). Pick **one** product root template: [`../assets/product-root-simple.md`](../assets/product-root-simple.md) (default) or [`../assets/product-root-jtbd.md`](../assets/product-root-jtbd.md) (job-centered outline—omit unused sections per template). See [`../SKILL.md`](../SKILL.md) defaults. **Do not mix** product-root styles in one coherent product tree.

## Draft session

1. **Load:** **B0** ([`../SKILL.md`](../SKILL.md) slice), **B1** (chosen product root template + for **JTBD**, [`jtbd-primer.md`](jtbd-primer.md)), **B3** (parent charter or repo context if any). Optional: [`context-loading.md`](context-loading.md).
2. **Draft** per template: **simple**—who, value, boundaries, outcomes; **JTBD**—executor / core job (optional step groups + outcome rows), optional related & emotional jobs, optional lifecycle & buyer, boundaries, outcomes index (one-liners)—**omit** sections that add no value.
3. **Emit** per [`emit-records.md`](emit-records.md): **CR** if meaningful edit; **PDR** only for a **new** product/strategic decision.

## AI review session

1. **Load:** **B2** [`review-product.md`](review-product.md) (product root section), **B1**, draft README. For **JTBD**, include [`jtbd-primer.md`](jtbd-primer.md).
2. **Review** against criteria; incorporate feedback.
3. **Emit** records if not already done.

## Second human review

Load **B2**, final README, short parent context—[`context-loading.md`](context-loading.md).
